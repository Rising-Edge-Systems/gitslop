## Codebase Patterns
- Tests live in `src/main/__tests__/*.test.ts` using vitest; run with `npm test` or `npx vitest run <file>`.
- Quality gates: `npm run typecheck` (node + web) and `npm test`. Both must be green before committing.
- Main-process code uses Node built-ins freely (crypto, http) — avoid new deps.
- Main process entry: `src/main/index.ts`. New main-process modules should be siblings of index.ts.
- Fixed loopback OAuth redirect: `http://localhost:47823/callback` (see `src/main/gitlab-oauth.ts` `LOOPBACK_PORT`). Do not change — it is baked into user-registered GitLab OAuth apps.
- GitLab auth headers differ by account type: PAT uses `PRIVATE-TOKEN: <token>`; OAuth uses `Authorization: Bearer <access_token>`. `gitlabApiRequest()` in `src/main/index.ts` currently only sends `PRIVATE-TOKEN` — US-GL-003+ must branch on `account.authType` when calling any `/api/v4/*` endpoint.
- `IntegrationAccount` in `src/main/index.ts` has optional OAuth fields: `authType?: 'pat' | 'oauth'`, `refreshToken?` (encrypted), `expiresAt?` (ms), `clientId?`. Missing/undefined `authType` means legacy PAT account.
