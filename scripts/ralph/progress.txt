## Codebase Patterns
- Sidebar: plain CSS flex div (NOT react-resizable-panels), width in pixels via layout.sidebarSize, drag handle in AppLayout
- Detail panel: plain CSS flex div, fixed 340px width (to be made resizable), always visible when repo open
- Right panel: DetailPanel (flex:6) + StatusPanel (flex:4) stacked vertically in a 340px column
- Staging area (StatusPanel): renders in the RIGHT panel, not center
- Center panel: commit graph OR diff viewer (swapped via viewingDiff state)
- DiffViewer: local useState for mode ('inline'|'side-by-side'), accepts initialMode prop, used in RepoView
- Sidebar collapsed: icon rail (48px) with RAIL_SECTIONS icons, overlay floats via overflow:visible + z-index on container
- Sidebar container in AppLayout: TWO instances (left + right position) — always update both
- CSS modules: import styles from './Component.module.css', camelCase class names
- Design tokens: --space-*, --font-*, --radius-*, --border, --bg-hover, --transition-fast
- Typecheck: npm run typecheck
- Build: npx electron-vite build && npx electron --no-sandbox .
- Layout state: useLayoutState() hook, localStorage key 'gitslop-layout-state', 300ms debounced save
- LayoutState fields: sidebarSize, bottomPanelSize, bottomPanelVisible, sidebarVisible, sidebarCollapsed, rightPanelSize (unused!), stagingCollapsed, detailPanelCollapsed
- Drag handle pattern: sidebar uses isDraggingRef + mousedown/mousemove/mouseup on document + body class 'sidebar-dragging'
- IPC pattern: git-service.ts method → git-ipc.ts handler → preload/index.ts expose → useLayoutState.ts type declaration
- DetailPanel hooks: ALL hooks must be above any early return to avoid React error #310
- CommitFileDetail type: { path, status, insertions, deletions, oldPath? } from CommitGraph.tsx
