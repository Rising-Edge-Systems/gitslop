## Codebase Patterns
- Sidebar is a plain `<div>` outside react-resizable-panels — never wrap it in a `<Panel>`
- `sidebarSize` in useLayoutState stores pixel width (260 default), not percentage
- Typecheck: use `npm run typecheck` (runs both `typecheck:node` and `typecheck:web`)
- `loadLayout()` in useLayoutState.ts migrates old percentage values (< 100) to 260px default
- AppLayout uses `layout.sidebarSize` for the sidebar div width; `setSidebarSize` clamps to 180-400px
- Sidebar drag handle: `.sidebar-drag-handle` in global.css, `body.sidebar-dragging` for cursor override during drag
- Sidebar wrapper div always renders with transition; collapsed=48px, expanded=sidebarSize. Transition disabled during drag via `isDragging` state
- `.iconRail` border-right removed — parent wrapper div handles the border
- Layout state props thread: AppLayout → MainContent → RepoView → StatusPanel (useLayoutState only in AppLayout)
