# Ralph Progress Log
Started: 2026-04-13

## Codebase Patterns
- Typecheck: `npm run typecheck` (runs both `typecheck:node` and `typecheck:web`)
- Tests: `npm test` (vitest run, 127 tests across 6 files)
- No lint script configured
- CommitGraph.tsx is ~2100 lines with all types, state, and rendering in one file
- IPC pattern: `window.electronAPI.git.<method>(repoPath)` returns `{ success: boolean, data?: T, error?: string }`
- Status IPC returns `{ staged: FileStatus[], unstaged: FileStatus[], untracked: FileStatus[] }` — count array lengths for summary
- Repo change events: `window.electronAPI.onRepoChanged()` for file watcher, `graph:force-refresh` custom event for post-operation refresh
- Lane assignment and graph layout via `assignLanes()` / `compactLanes()` from `./laneAssignment`
- Commit selection: `handleRowClick`, `handleCanvasNodeClick`, keyboard handler all manage selectedIndex/selectedHash
- WIP row uses wipOffset pattern: `wipOffset = wipStatus ? 1 : 0` — all indices in handlers, SVG, Canvas must add wipOffset to node index for Y-position calculation
- CommitRowProps extends with wipStatus/wipOffset/isWipSelected — row component checks `index === 0 && wipOffset > 0` for WIP rendering
- Both GraphSVG and GraphCanvas need wipOffset prop to offset node positions and render WIP node
