## Codebase Patterns
- Typecheck: `npm run typecheck` (runs typecheck:node + typecheck:web)
- CommitGraph.tsx uses react-window v2 with `useListCallbackRef` for list virtualization
- CSS modules pattern: `styles.className` with `CommitGraph.module.css`
- HEAD commit identified by `refs.some(r => r.type === 'head')` — type 'head' means the current branch ref
- Lane 0 is always HEAD lane; lane colors via `color-mix()` CSS function
- Floating overlays: position absolute within `.container`, z-index 1200+ range
- `listRef.scrollToRow({ index, align })` for programmatic scrolling
- Graph nodes computed via `computeGraphLayout(commits)` → `useMemo`
- All repo-modifying git ops go through `withWatcherSuppression()` in git-ipc.ts — it handles watcher suppression + forced refresh
- `sendRepoChangedForced()` in index.ts bypasses watcher suppression for explicit repo:changed events

- Files changed: src/main/index.ts, src/main/git-ipc.ts
- **What was done:**
  - Added `sendRepoChangedForced()` in index.ts: sends `repo:changed` IPC event bypassing watcher suppression (150ms delay for disk writes)
  - Modified `withWatcherSuppression()` in git-ipc.ts to call `sendRepoChangedForced()` after every successful git operation
  - All operations using `withWatcherSuppression` (merge, commit, checkout, push, pull, fetch, rebase, cherry-pick, revert, reset, stash, tag, branch ops) now trigger immediate graph refresh
  - Scroll position preserved naturally: `loadCommits` only updates data array, react-window maintains position
- **Learnings for future iterations:**
  - `withWatcherSuppression` is the central wrapper for ALL repo-modifying git ops in git-ipc.ts — modifying it affects everything
  - The watcher suppression pattern: suppresses during ops + 1s after → file change events are dropped → need explicit event after completion
  - `sendRepoChanged()` is private and checks suppression; `sendRepoChangedForced()` is exported and bypasses it
  - CommitGraph already listens to `repo:changed` via `onRepoChanged` and calls `loadCommits()` — no renderer changes needed
