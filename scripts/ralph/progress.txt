- Renderer components are in `src/renderer/src/components/`
- Use `lucide-react` for icons: `import { IconName } from 'lucide-react'` with `size` and `className` props
- Toolbar emoji icons use `<span className="toolbar-btn-icon">` wrapper pattern
- Typecheck: `npm run typecheck` (runs both node and web)
- PRD branch is `ralph/ui-overhaul`, created from `ralph/gitslop-v1`
- SidebarSection `icon` prop is `React.ReactNode` — pass icons as `icon={<IconName size={16} />}`

- All emoji/unicode icons replaced with Lucide as of US-042c — zero emoji/unicode remaining in renderer

- tokens.css is imported BEFORE global.css in main.tsx — all color/spacing/typography vars live there
- CSS modules: `import styles from './Component.module.css'` then `className={styles.myClass}` — type declarations in `src/renderer/src/css-modules.d.ts`
- CSS module class names use camelCase (e.g., `styles.btnActive`) instead of kebab-case
- Shared dialog styles (`branch-dialog-*`) stay in global.css — don't move them into component modules
- `branch-dialog-btn-primary`, `branch-dialog-btn-secondary`, `branch-dialog-btn-danger` are shared across Toolbar, Sidebar, MergeDialog, RebaseDialog, SearchPalette
- `branch-ctx-menu-*` styles are shared across Sidebar, FileTree, ContextMenu — keep in global.css
- FileTree.tsx imports sidebarStyles for shared classes (placeholder, searchBox, searchInput, searchClear) — update FileTree when modifying sidebar styles
- ALL component styles are now in CSS modules — global.css has only reset, layout, shared branch-dialog/ctx-menu, resize handles, syn-* tokens, and slideUp animation
- Cross-component CSS module imports are fine: e.g., RepoView imports blameStyles, conflictStyles, editorStyles for child component wrappers
- File watcher: all mutating git IPC handlers are wrapped with `withWatcherSuppression()` from git-ipc.ts; watcher ignores ALL `.git/` and `node_modules/` changes
- global.css is ~448 lines (8.5KB) — the branch-dialog/ctx-menu shared styles are the bulk
- Three-column layout: Group(vertical) > [Group(horizontal) > Sidebar | Center | DetailPanel] | Terminal — terminal spans full width
- `onCommitSelect` callback chain: AppLayout → MainContent → RepoView → CommitGraph — lifts selected commit to layout level
- CommitDetail type exported from CommitGraph.tsx — use for detail panel props
- Layout state (useLayoutState) persists sidebar, bottom panel, and right panel sizes to localStorage
- Sidebar collapse: `sidebarCollapsed` in layout state, icon rail rendered outside resizable panels as fixed 48px flex item
- Sidebar props: `collapsed: boolean` and `onToggleCollapse: () => void` — both required
- `useWindowWidth()` hook tracks window.innerWidth — use for responsive breakpoints (e.g., DETAIL_PANEL_BREAKPOINT = 1400px)
- Detail panel overlay: `overlay` prop switches between inline (resizable column) and overlay (fixed + backdrop) modes
- Lane assignment: `assignLanes()` from `laneAssignment.ts` — pure function, takes `LaneCommit[]`, returns `LaneAssignmentResult[]` with lane, color, parentConnections
- Data refresh: components use `window.electronAPI.onRepoChanged()` with debounced timeouts (500-2000ms), NOT setInterval polling
- Loading state: only show loading spinner on initial mount (track via `initialLoadDone` ref), refresh silently in background
- In-flight guards: use refs (`refreshInFlightRef`) to prevent overlapping async calls in the same component
- MainContent is wrapped in React.memo — avoid passing new object/function references as props unless necessary
- SVG commit graph: `GraphSVG` component replaces canvas; uses `assignLanes()` from laneAssignment.ts; `node.color` is pre-computed per lane
- GraphNode uses `lane` (not `column`) and `parentConnections` (not `parents`) — aligned with laneAssignment.ts types
- All context menus use the reusable `ContextMenu` component — build items with `ContextMenuEntry[]` arrays, not custom menu components
- `branch-ctx-menu-*` CSS has been removed from global.css — all context menus now use ContextMenu.module.css styles
