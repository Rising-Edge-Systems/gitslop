## Codebase Patterns
- Use `electron-vite` v4 with `vite` v6 (v5+ has peer dep conflicts with electron-vite v5)
- Project uses separate tsconfigs: `tsconfig.node.json` (main+preload), `tsconfig.web.json` (renderer)
- Run `npm run typecheck` to check both node and web TypeScript
- electron-vite v4 doesn't export `is` from `electron-vite/node`; use `!app.isPackaged` instead for dev detection
- Build output goes to `out/` directory (electron-vite default)
- NVM must be sourced before using node: `export NVM_DIR="$HOME/.nvm" && [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"`
- Custom titlebar with frameless window; IPC channels prefixed with `window:`, `dialog:`, `git:`, `repos:`
- Global `Window` interface with `electronAPI` is declared in `src/renderer/src/hooks/useLayoutState.ts` (single declaration)
- electron-store used in main process for persisting recent repos (schema typed with `Store<StoreSchema>`)
- `RecentRepo` type exported from `useLayoutState.ts` for renderer use
- CSS uses custom properties (CSS variables) defined in `:root` in `src/renderer/src/styles/global.css`
- React components go in `src/renderer/src/components/`
- Custom hooks go in `src/renderer/src/hooks/`
- `react-resizable-panels` v4.7+ uses `Group`, `Panel`, `Separator` (not `PanelGroup`/`PanelResizeHandle`)
- `react-resizable-panels` v4 `onResize` callback receives `PanelSize` object with `.asPercentage` and `.inPixels`
- Layout state persisted via `localStorage` with key `gitslop-layout-state`
