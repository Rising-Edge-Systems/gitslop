## Codebase Patterns
- Typecheck: use `npm run typecheck` (not bare `tsc --noEmit`), it runs both `typecheck:node` and `typecheck:web` with correct tsconfigs
- Tests: `npm test` runs vitest (103 tests currently)
- Custom events pattern: use `window.dispatchEvent(new CustomEvent('event-name'))` and `window.addEventListener('event-name', handler)` for cross-component communication in renderer
- Git operations in Toolbar.tsx: push (handlePush, handleForcePush, handleSetUpstream), pull (handlePull, handlePullConfirm), fetch (handleFetch) — all use `window.electronAPI.git.*`
- File watcher: `window.electronAPI.onRepoChanged` fires on file changes; `withWatcherSuppression` in git-ipc.ts wraps operations and calls `sendRepoChangedForced()` after
- CommitGraph.tsx loadCommits() has an optimization that skips re-render if first+last hash+refs match — force refresh bypasses this
- Window.electronAPI global type declaration is in `src/renderer/src/hooks/useLayoutState.ts` — must update there when adding new IPC methods
- IPC method flow: git-service.ts (impl) → git-ipc.ts (handler) → preload/index.ts (bridge) → useLayoutState.ts (type decl)
