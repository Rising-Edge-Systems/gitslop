## Codebase Patterns
- IPC pattern: git-service.ts method -> git-ipc.ts handler -> preload/index.ts expose -> useLayoutState.ts type declaration (4 files for each new git operation)
- Existing IPC: showFileAtCommit runs `git show HASH:PATH` — follow same pattern for showFileAtParent
- DiffViewer.tsx: has InlineDiffView and SideBySideDiffView internal components, mode toggled via useState
- DiffViewMode type: currently 'inline' | 'side-by-side', exported from DiffViewer.tsx and useLayoutState.ts
- Center diff rendering: RepoView.tsx renders DiffViewer when viewingDiff=true, also handles 'file' mode with a separate fetch
- Side-by-side view: SideBySideDiffView has scroll sync between left/right panes via refs and onScroll handlers
- DiffHunk type: { header, headerSuffix, lines[] } where each line has { type: 'added'|'removed'|'context', content, oldLineNum?, newLineNum? }
- Syntax highlighting: SyntaxHighlightedContent component and highlightLine function
- CSS modules: import styles from './DiffViewer.module.css'
- sbsPane/sbsPaneInner/sbsPaneHeader classes for side-by-side pane structure
- Binary detection: check for null bytes (\0) in content
- Typecheck: npm run typecheck
- Build: npx electron-vite build && npx electron --no-sandbox .
- Center view mode in RepoView.tsx: 'diff' | 'full' | 'file' — separate from DiffViewer's inline/split toggle
- FullDiffView component is co-located in DiffViewer.tsx to share parseDiff, detectLanguage, SyntaxHighlightedContent
- DiffViewMode: 'inline' | 'side-by-side' | 'full' | 'file' — persisted in LayoutState, used by RepoView to derive centerViewMode
- CommitFileDetail: has oldPath for renames, status codes M/A/D/R/C
