## Codebase Patterns
- DiffViewer.tsx: InlineDiffView and SideBySideDiffView are internal components
- InlineDiffView: renders inside div.inlineView (overflow: auto), contains div.inlineViewInner (inline-block)
- SideBySideDiffView: renders inside div.sbsView with two div.sbsPane children, each has div.sbsPaneInner
- FullFileDiffView: similar to SideBySideDiffView but shows complete files, inside its own container
- DiffHunk type: { header, headerSuffix, lines[] } where line has { type, content, oldLineNum?, newLineNum? }
- Line types: 'added' | 'removed' | 'context' | 'meta'
- Scroll sync in SBS: leftPaneRef/rightPaneRef with onScroll handlers that sync scrollTop
- CSS modules: styles from './DiffViewer.module.css'
- The scroll container for inline view is .inlineView, for SBS is each .sbsPane
- Typecheck: npm run typecheck
- Build: npx electron-vite build && npx electron --no-sandbox .

## 2026-04-09 - US-SM-001
- Implemented ScrollbarMarkers component: renders colored markers on a thin absolute-positioned column on the right edge of diff scroll containers
- Added `computeMarkers()` utility that merges consecutive changed lines into single marker entries with proportional position/height
- Integrated markers into InlineDiffView (wraps .inlineView in .scrollableWithMarkers) and FullDiffView (wraps each .sbsPane)
- Click-to-scroll: clicking anywhere on the marker column scrolls the container to that proportional position
- Files changed: DiffViewer.tsx, DiffViewer.module.css
- **Learnings for future iterations:**
  - The `scrollableWithMarkers` wrapper must have `position: relative; flex: 1; overflow: hidden` to work correctly in both flex column (inline) and flex row (SBS) layouts
  - When wrapping `.sbsPane` in FullDiffView, the wrapper takes on the flex child role so it needs `flex: 1; min-width: 0`
  - The markers use `clamp(minPx, %, maxPx)` for height to ensure visibility for single lines while capping large hunks
  - `computeMarkers` merges consecutive same-type lines into single markers for efficiency
